The 15-location lead sync

"Turned a weekly export ritual into a system that maintains itself."

Client: Northstar Preschools Stack: Google Sheets · Apps Script · CRM API Result: 5+ hrs/week saved

Problem Reporting was always a week behind

Northstar Preschools, a 15-location preschool network, tracked leads in a CRM, but reporting lived in Google Sheets. Getting data from one to the other meant manual exports: download, clean, paste, repeat. Every week.

The cost wasn't just the hours. Exports went stale the moment they were made, duplicates crept in, and nobody fully trusted the numbers, so decisions kept waiting on "the latest version".

Build A sync that runs itself, and repairs itself

I built an Apps Script integration that pulls leads straight from the CRM's API into a master workbook, continuously:

  • One raw mirror, 15 location views. A single source-of-truth tab feeds a colour-coded tab per location, newest leads on top, each row linked back to the lead in the CRM.
  • A permanent status history. The CRM only shows a lead's current status. The sync logs every status change, building a history the CRM itself doesn't keep.
  • Batched writes. Apps Script has hard execution limits; writes are batched so the sync never times out, no matter the volume.
  • Deduplication and self-repair. The script detects broken or duplicated rows and fixes them on its own. Nobody babysits it.
The result

5+ hours of manual exporting eliminated every week. One source of truth, updated automatically, trusted by leadership.
